# 微信公众号内容分析

## 概述

本技能用于对微信公众号（Official Account）内容进行多维度深度分析，涵盖内容质量评估、互动数据分析、写作风格识别、内容策略诊断与优化建议。适用于运营人员、内容创作者、品牌营销团队及行业研究者。

当需要详细的评分标准与行业基准数据时，查阅 `references/analysis-rubric.md`；当需要报告模板与输出格式规范时，查阅 `references/report-templates.md`。

## 适用场景

- **单篇文章深度分析**：用户粘贴或提供公众号文章链接/文本，需评估内容质量、结构、可读性
- **账号整体诊断**：用户希望了解某个公众号的内容定位、发布节奏、爆款特征
- **多账号对比分析**：用户需要横向对比 2 个及以上公众号的表现差异
- **内容策略制定**：用户希望基于数据获得选题、标题、发布时间等优化建议
- **行业内容趋势研究**：用户需要识别某领域公众号的内容趋势与热点

## 分析维度

### 一、内容质量分析

#### 1. 标题分析
- 标题类型识别：资讯型 / 悬念型 / 数字型 / 情感型 / 对比型 / 盘点型 / 痛点型
- 吸引力评分（1-10 分）
- 关键词提取与 SEO 价值
- 标题党特征检测（过度夸张、信息隐藏、情绪煽动）
- 长度适宜性评估（建议 16-22 字，朋友圈折叠线前需完整展示）

#### 2. 结构分析
- 开头策略：故事引入 / 提问 / 数据冲击 / 结论先行 / 场景描写
- 段落层次与逻辑递进
- 小标题使用（是否清晰、是否有层级）
- 图文配比与排版节奏
- 结尾引导设计（CTA：关注/点赞/在看/留言/转发）
- 整体字数与预估阅读时长（按 400 字/分钟估算）

#### 3. 可读性评估
- 平均句长（建议 15-25 字/句）
- 专业术语密度（受众匹配度）
- 信息密度（干货比例）
- 口语化 vs 书面化程度
- 视觉留白与阅读节奏

#### 4. 主题与情感
- 核心主题识别与分类
- 情感倾向判定（积极 / 中性 / 消极 / 焦虑诱导）
- 价值观传递与品牌调性一致性

### 二、互动数据分析

当用户提供互动数据时（阅读量、点赞、在看、评论、分享）：

#### 1. 核心指标计算
- 阅读量：绝对值 + 相对账号基线
- 互动率 = (点赞 + 在看 + 评论 + 分享) / 阅读量 × 100%
- 分项互动率：点赞率、在看率、评论率、分享率

#### 2. 基准对比
参照 `references/analysis-rubric.md` 中的行业基准：
- 头部账号互动率参考：3-8%
- 中腰部账号：1-3%
- 长尾账号：< 1%
- 爆款识别阈值：互动率 > 账号平均 2 倍

#### 3. 内容-互动关联
- 高互动内容主题特征提取
- 标题类型与阅读量关系
- 发布时间与互动表现关系
- 内容长度与完读率（在看率可作完读率代理指标）

### 三、内容策略分析

#### 1. 发布节奏
- 更新频率（日更 / 周更 / 月更 / 不定期）
- 发布时间分布与黄金时段识别
- 黄金时段参考：早 7-9 点（通勤）、午 12-14 点（午休）、晚 20-22 点（睡前）

#### 2. 内容矩阵
- 主题分布占比（以饼图或表格呈现）
- 内容类型分布：干货教程 / 行业资讯 / 故事情感 / 活动推广 / 互动话题 / 转载
- 原创比例与转载来源

#### 3. 爆款规律
- 爆款内容主题共性
- 爆款标题特征聚类
- 爆款发布时间分布
- 爆款与日常内容差异点

### 四、账号定位诊断

- **定位清晰度**：内容是否聚焦单一领域或人群
- **人设一致性**：语言风格、视角、价值观是否统一
- **差异化程度**：与同类账号的核心区别
- **用户价值**：提供的信息价值 / 情感价值 / 服务价值 / 娱乐价值

## 分析工作流

### 步骤 1：明确分析目标
向用户确认：
- 分析对象（单篇文章 / 单个账号 / 多个账号）
- 分析目的（诊断 / 优化 / 对比 / 趋势研究）
- 是否有互动数据可提供
- 期望输出形式（可视化报告 / 简明清单 / 对话总结）

### 步骤 2：数据获取
根据数据来源选择获取方式：
- **文本类**：用户直接粘贴文章正文
- **链接类**：`mp.weixin.qq.com` 链接，使用浏览器工具抓取正文与元数据
- **数据类**：用户提供截图或表格数据（阅读量、互动数等）
- **截图类**：使用图像理解能力解析公众号后台截图

### 步骤 3：执行分析
按上述"分析维度"逐项执行，每项输出需包含三层：
1. **客观描述**：是什么（数据/事实陈述）
2. **评分判定**：好不好（1-10 分 + 等级）
3. **证据引用**：哪里看出来（原文引用或数据支撑）

### 步骤 4：生成洞察
- 3-5 条核心发现，标注 Strengths（优势）/ Weaknesses（不足）
- 优先级排序的优化建议（按影响力 × 可行性排序）
- 可执行的下一步行动清单

### 步骤 5：输出报告
根据用户期望选择输出格式，具体模板参照 `references/report-templates.md`：
- **结构化报告**：调用 `html-report` 技能生成可视化 HTML 报告
- **简明清单**：Markdown 表格 + 要点列表
- **对话式总结**：直接在对话中呈现核心结论

## 输出规范

### 评分体系
各维度采用 1-10 分制：
- 9-10 分：行业标杆
- 7-8 分：优秀
- 5-6 分：合格
- 3-4 分：待优化
- 1-2 分：严重不足

### 语言风格
- 客观、数据驱动，避免主观臆断
- 避免空泛赞美（如"写得很好"），必须给出具体证据
- 建议需可执行，避免"加强内容质量"类空话，应写明"将标题字数控制在 18 字以内，前 8 字包含核心关键词"
- 引用原文时标注出处段落

## 注意事项

1. **数据真实性**：微信互动数据存在刷量可能，分析时应提示用户注意数据可信度
2. **时效性**：公众号算法与功能频繁更新（如推荐机制改版），分析前确认当前规则
3. **合规性**：分析建议不得涉及诱导分享、刷量、标题党等违反微信运营规范的行为
4. **隐私保护**：仅分析公开可见内容，不采集或分析个人用户隐私数据
5. **行业差异**：不同行业公众号的基准数据差异大（如财经类阅读量普遍高于教育类），避免一刀切评判

## 示例用法

### 示例 1：单篇文章分析
**用户**："帮我分析这篇文章 https://mp.weixin.qq.com/s/xxxxx"

**执行流程**：
1. 使用浏览器工具抓取文章正文、标题、作者、发布时间
2. 按内容质量四维度（标题/结构/可读性/主题情感）逐项分析
3. 输出评分卡表格 + 3 条优化建议

### 示例 2：账号诊断
**用户**："分析一下『XX财经』这个公众号最近的内容策略"

**执行流程**：
1. 收集该账号近期 10-20 篇文章样本
2. 完成内容矩阵分析（主题分布、类型分布、发布节奏）
3. 识别爆款规律与定位清晰度
4. 生成诊断报告 + 策略优化建议

### 示例 3：多账号对比
**用户**："对比这两个教育类公众号的内容表现"

**执行流程**：
1. 分别收集两个账号的样本文章与互动数据
2. 按统一维度完成各自分析
3. 生成对比表格（维度 × 账号）
4. 给出差异化定位建议
