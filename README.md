# 微信公众号内容分析技能

一个帮你把公众号文章拆开揉碎了看的内容分析方法库。不是简单的"好/不好"评价，而是给出可量化的评分、可复用的结构拆解、可执行的改进建议。

适合做公众号运营、品牌内容、新媒体投放分析、竞品研究的人用。无论你是要复盘自己号的某篇爆款、诊断某个账号的整体内容策略，还是横向对比同领域几个竞品号，这个技能都能给出基于数据和事实的判断，而不是凭感觉说"这篇写得不错"。

## 这个技能能干什么

把一篇公众号文章丢给它，它会从四个角度拆：

### 标题分析
识别标题属于哪一类型（资讯型 / 悬念型 / 数字型 / 情感型 / 对比型 / 盘点型 / 痛点型），给吸引力打 1-10 分，检测有没有标题党特征（过度夸张、核心信息隐藏、情绪煽动、与正文不符），评估长度是否合适（建议 16-22 字，朋友圈折叠线前要能完整展示核心信息）。

### 结构分析
看开头怎么切入的（故事引入 / 提问 / 数据冲击 / 结论先行 / 场景描写），正文段落层次和逻辑递进，小标题用得怎样，图文配比和排版节奏，结尾有没有引导互动（点赞 / 在看 / 留言 / 转发），整体字数和预估阅读时长（按 400 字/分钟估）。

### 可读性评估
算平均句长（建议 15-25 字），看专业术语密度是不是匹配受众，信息密度（干货比例），口语化还是书面化，视觉留白和阅读节奏。

### 主题与情感
识别核心主题和分类，判定情感倾向（积极 / 中性 / 消极 / 焦虑诱导），看价值观传递和品牌调性是不是一致。

## 如果你有互动数据

提供阅读量、点赞、在看、评论、分享这些数据时，技能会做更深入的分析：

- 算互动率 = (点赞 + 在看 + 评论 + 分享) / 阅读量 × 100%
- 分项互动率：点赞率、在看率、评论率、分享率
- 和行业基准对比（头部 3-8%、中腰部 1-3%、长尾 <1%）
- 识别爆款（互动率超过账号均值 2 倍的）
- 看内容主题、标题类型、发布时间和互动表现之间的关联

## 如果你给一批文章

给某个号近期 10-20 篇文章样本，技能能看到更整体的东西：

- **内容矩阵**：主题分布占比、内容类型分布（干货教程/行业资讯/故事情感/活动推广/互动话题/转载）、原创比例
- **发布节奏**：更新频率、发布时间分布、黄金时段命中率
- **爆款规律**：爆款在主题、标题、发布时间上的共性，和日常内容的差异点
- **定位诊断**：定位清晰度、人设一致性、差异化程度、用户价值

## 仓库内容

```
├── SKILL.md                       # 技能主文件，AI 助手按这个执行分析
└── references/
    ├── analysis-rubric.md         # 评分细则和行业基准数据
    └── report-templates.md        # 三种报告模板（单篇/账号诊断/多账号对比）
```

`analysis-rubric.md` 里有详细的 1-10 分判定标准、标题党特征检测清单、各行业阅读量参考、黄金时段表、内容矩阵健康度指标。打分的时候对照这里，避免凭感觉。

`report-templates.md` 里有三种报告的完整结构模板：单篇文章分析报告、账号诊断报告、多账号对比报告。每种都含表格结构、字段定义、输出格式选择指南。

## 怎么用

### 场景一：分析单篇文章

粘贴 `mp.weixin.qq.com` 链接或正文，告诉它"分析一下"。它会：

1. 抓取文章元数据（标题、作者、发布时间、字数）
2. 按四个维度逐项打分，每项给 1-10 分加等级
3. 每个评分都附原文引用作为证据，不空泛评价
4. 输出评分卡表格和 3 条可执行的改进建议

示例输出结构：

```
## 评分总览
| 维度 | 评分 | 等级 |
|------|------|------|
| 标题吸引力 | 7/10 | 优秀 |
| 结构逻辑 | 5/10 | 合格 |
| 可读性 | 6/10 | 合格 |
| 主题与情感 | 8/10 | 优秀 |
| 综合评分 | 6.5/10 | 合格 |

## 优化建议（按优先级）
1. 将标题字数从 28 字压缩到 18 字以内，前 8 字包含核心关键词
2. 正文第 3 段长达 420 字，拆分为 2-3 段并加小标题
3. 结尾缺少互动引导，增加"评论区说说你的看法"类 CTA
```

### 场景二：诊断某个账号

提供该号近期 10-20 篇文章样本（链接或正文），技能会：

- 统计主题分布、内容类型分布、发布频率和时段
- 识别爆款文章，分析它们在主题、标题、发布时间上的共性
- 评估定位清晰度（内容是否聚焦单一领域或人群）
- 给出内容矩阵健康度诊断（主题聚焦度 >70%、原创比例 >60% 等指标）
- 输出诊断报告和按优先级排序的优化建议

### 场景三：对比多个账号

给两个以上账号的样本和数据，技能会：

- 按统一维度（内容质量、互动表现、内容策略、定位）各自分析
- 出对比表格（维度 × 账号）
- 分析两个账号的核心差异和各自优势领域
- 给出差异化定位建议，避免同质化竞争

## 评分体系

各维度 1-10 分：

| 分数区间 | 等级 | 含义 |
|---------|------|------|
| 9-10 | 行业标杆 | 值得其他账号学习 |
| 7-8 | 优秀 | 保持现状，小幅优化 |
| 5-6 | 合格 | 有明显改进空间 |
| 3-4 | 待优化 | 存在结构性问题 |
| 1-2 | 严重不足 | 需要重新思考方向 |

所有评分必须附证据引用（原文段落或数据支撑），不接受"写得很好"这种空泛评价。改进建议必须可执行，"加强内容质量"这种空话会写成"将标题字数控制在 18 字以内，前 8 字包含核心关键词"。

## 行业基准参考

不同行业的基准数据差异很大，技能内置了分行业参考：

| 行业 | 头部账号阅读量 | 中腰部账号阅读量 |
|------|--------------|----------------|
| 财经 | 10万+ | 1-5万 |
| 科技 | 5-10万 | 5000-2万 |
| 教育 | 3-5万 | 3000-1万 |
| 时尚美妆 | 5-10万 | 5000-2万 |
| 本地生活 | 1-3万 | 1000-5000 |
| 企业官号 | 5000-2万 | 500-3000 |

完整的互动率基准、分项互动率健康范围、发布频率健康度、黄金时段表在 `references/analysis-rubric.md`。

## 几个注意点

- 微信互动数据有刷量可能，分析时会给数据可信度提示，不会盲目相信数字
- 不同行业的基准差异很大（财经号阅读量天然比教育号高），不会拿同一把尺子量所有人
- 建议不涉及诱导分享、刷量、标题党这类违反微信运营规范的行为
- 只分析公开可见内容，不采集或分析个人用户隐私数据
- 公众号算法和功能更新频繁（比如推荐机制改版），分析前会确认当前规则
